# Roadmap & Milestones (suggested 12-week MVP)

## Phase 0: Preparation (Weeks 0–1)

- lock MVP scope + “high-risk not-do” list (e.g., housing marketplace matchmaking)
- finalize IA (city/campus + section + topics)
- wireframes + baseline visual system
- finalize content templates (housing/scams/transport/groceries)

Deliverables:

- prototype + page list
- draft data model
- moderation + governance rules draft

## Phase 1: Content loop (Weeks 2–4)

- auth + roles (user/contributor/moderator)
- content publishing (guides/experience posts with templates)
- basic moderation queue (pending/approved/rejected/needs info)
- search & filters (city/campus/topics)
- reporting + basic anti-spam (rate limits, link policy)

Deliverables:

- closed loop: publish + review + report
- housing/scams sections MVP

## Phase 2: AI Q&A MVP (Weeks 5–6)

- build reviewed-content knowledge base (embeddings + metadata)
- RAG Q&A: answer + citations + trust label + risk warnings
- Q&A UX: clarifying questions, save/share, correction entry

Deliverables:

- usable AI Q&A grounded in reviewed content
- answer logging + basic monitoring

## Phase 3: AI moderation + risk controls (Weeks 7–8)

- AI pre-screen: illegal/privacy/ads/scams tags
- redaction suggestions + author rewrite guidance
- mandatory human review for high-risk content (transfers/contacts/non-official links)

Deliverables:

- machine pre-screen + human review linkage
- internal dashboards for hit rate / false positives

## Phase 4: Volunteer support (Weeks 9–10)

- volunteer application + review
- ticket-based help (anonymous, category, language, time window)
- assignment + SLA + closure feedback

Deliverables:

- controlled “real human support” loop
- safety guidelines + boundaries

## Phase 5: Pilot & ops (Weeks 11–12)

- campus/city pilot (seed content + volunteers)
- staleness reminders + update mechanism
- metrics + growth experiments (subscriptions, sharing, recommendations)

Deliverables:

- pilot launch
- reusable ops SOP (content updates, moderation SLA, scam alerts)
