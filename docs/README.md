# Bridge US — 项目文档目录

本目录用于存放项目的产品与技术规划文档，便于团队对齐目标、范围、节奏与质量标准。

## 文档导航

- `01_项目概览.md`：一句话定位、愿景、核心原则、差异化
- `02_主要目的与问题定义.md`：项目主要目的、要解决的真实问题、成功的定义
- `03_面向对象与用户场景.md`：目标用户、使用场景、用户旅程与痛点
- `04_功能规划与信息架构.md`：模块拆分、信息架构、优先级（MVP/增强）
- `05_内容可信与社区治理.md`：无广告原则、审核流程、反垃圾/反诈骗、安全与隐私
- `06_AI_与人审结合方案.md`：AI 问答/RAG/来源引用/可信标记、AI 审查与风控、边界与责任
- `07_技术架构建议.md`：技术栈、服务划分、数据模型概览、可观测性与安全
- `08_路线图与里程碑.md`：分阶段交付计划（按周/按版本）
- `09_指标与运营策略.md`：北极星指标、质量指标、增长与校园试点策略
- `10_MVP_初始版本_FastAPI_功能清单.md`：初始版本必须做的功能清单（以 FastAPI 后端为中心）
- `11_多语言与翻译策略.md`：全站中英双语支持、默认英文开发、帖子 AI 自动翻译工作流与数据结构
- `12_UI_设计规范.md`：现代、简约、高级的 UI 设计规范（颜色/字体/组件/页面布局）
