# 内容可信与社区治理

## 目标

在不牺牲用户体验的前提下，让平台内容尽可能做到：

- **少广告**：减少营销动机对内容的污染
- **可验证**：重要信息能追溯来源与证据
- **可更新**：内容会过期，平台要能推动更新与纠错
- **更安全**：减少诈骗、骚扰与隐私泄露

## 无广告伪装原则（核心差异化）

- **禁止赞助内容伪装成经验/建议**。
- 若存在任何商业利益关联，必须 **显著标注**（置顶标签 + 文末声明）。
- 对“外链/联系方式/引流”做强限制与风控（默认不展示或需要审核后展示）。

## 可信度体系（可解释、可执行）

建议对内容显示以下字段：

- **验证状态**：`已验证` / `部分验证` / `待验证`
- **更新时间**：最后一次更新日期
- **适用范围**：城市/学校/时间窗口（例如 2025 秋季）
- **来源/证据**：官方链接、截图（可脱敏）、票据（可选）

> 注：验证状态不等于“绝对正确”，而是对“可追溯程度与核验完成度”的标注。

## 审核流程（建议）

### 流程概览

1. 用户提交内容（结构化字段 + 可选证据）
2. 机器初筛（广告/敏感/诈骗关键词、外链策略、重复度检测）
3. 人工审核（志愿者/版主）
4. 发布并展示验证状态
5. 社区纠错与更新（提交补充 → 复审 → 更新记录）

### 审核队列建议

- **按主题队列**：住房/诈骗优先级最高
- **按风险评分排序**：外链、转账关键词、联系方式、极端承诺等提升风险
- **需要补充信息**：让作者补齐“地点/时间/证据/适用范围”

## 反垃圾与反诈骗

- **发帖频控**：新用户限流；异常行为触发冷却
- **相似度去重**：减少批量灌水与广告改写
- **外链策略**：白名单优先（官方/学校网站）；其它链接需要审核
- **关键词与模式库**：诈骗话术、转账诱导、虚假优惠等
- **用户举报闭环**：举报 → 分流（垃圾/诈骗/骚扰/隐私）→ 处理结果可追踪

## 安全与隐私默认策略

- 默认匿名显示（昵称化），严禁公开敏感信息（电话、住址门牌、证件号等）
- 图片上传自动处理：EXIF 清理；可选自动打码（电话/人脸/地址）
- 志愿者连接采用 **工单式**（不直接暴露联系方式），并提供安全提示与边界声明

## 社区激励机制（围绕“有用”而非“热度”）

- “**有用**”替代“点赞”
- 贡献积分来源：
  - 提供可复用经验（结构完整）
  - 通过审核并被标记为已验证/部分验证
  - 帮助更新过期内容
  - 参与核验/翻译
- 约束：积分不能直接兑换“曝光特权”导致刷分；需要风控与异常检测
