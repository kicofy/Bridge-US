# 功能规划与信息架构

## 信息架构（IA）

建议以“**地点（城市/学校）** + **板块（用户阶段/任务）** + **主题（细分标签）**”组织内容。

### 板块分类（一级入口，用户最容易理解）

> 说明：板块用于回答“我现在处在什么阶段/要完成什么事”，是首页与导航的主入口。

- **落地第一周**：电话卡/银行卡/交通卡、临时住宿、基本购物、校园报到、紧急联系人
- **找房与入住**：看房清单、合同与押金、室友沟通、治安与通勤、常见骗局与核验
- **食物与超市**：传统食材、超市/亚超地图、替代品、价格区间、省钱买法
- **交通出行**：公交/地铁/校园巴士、夜间安全、常用 App、学生折扣
- **办事与生活**：医保与医疗、证件与办事流程、报税、驾照、运营商/银行常见问题
- **安全与诈骗**：高危套路、预警、黑名单规则、报警与求助路径（内容严格审核）

（可选后置板块：省钱与二手、校园资源与社交、心理与适应）

- 首页
  - 城市/学校选择
  - 板块入口：落地第一周 / 找房与入住 / 食物与超市 / 办事与生活 …
  - 快速入口：落地 7 天清单 / 找房 30 天清单 / 反诈骗必读
  - AI 快速问答入口（显示来源与可信状态）
- 主题频道
  - 主题用于“细分标签/筛选”，不一定作为主导航（例如：电话卡、银行、驾照、保险、通勤、合同）
- 社区与互助
  - 经验帖/问答/评论
  - 志愿者求助（工单）
- 个人中心
  - 收藏/订阅标签
  - 贡献记录与积分
  - 举报/申诉记录

## 核心功能模块

### 内容（Guides + Experience）

- **指南/清单**：结构化内容（步骤、链接、注意事项、更新时间）
- **经验帖**：真实经历（可匿名），支持模板化字段提高可比性
- **标签与筛选**：城市/学校/板块（阶段）/主题标签/预算/通勤/安全/语言等

### 住房（高优先级）

- **真实住房体验**：租金区间、位置（可模糊到区域）、通勤、治安主观评分、合同注意事项
- **避坑与骗局案例**：常见套路、核验清单、转账风险提示
- **安全提示**：默认隐藏敏感信息；外链与联系方式受控展示

### 交通与生活

- 公交/地铁/校园巴士、常用 App、购票与折扣、夜间安全通勤建议
- 传统食材与超市：按地区、价位、可买到的品类、替代品

### 志愿者连接（受控）

- 工单式求助：类别、时间窗、语言、匿名需求
- 志愿者审核与匹配：角色认证、服务边界声明、违规处理

### 质量与治理（平台底座）

- 举报/申诉
- 审核队列与证据管理
- 反垃圾/反广告/反诈骗
- 过期内容提醒与更新激励

## MVP 与迭代优先级

### MVP（先做能真正解决问题的）

- 城市/学校 + 板块（阶段）维度浏览（比“主题”更好用）
- 指南/经验帖发布（含模板字段）
- “有用”反馈与收藏
- 基础审核流程（待审/已审/驳回/需补充）
- 诈骗与安全专题页
- AI 问答 MVP：仅检索“已审核内容”，答案展示来源与可信标记

### 迭代增强（第二阶段）

- 志愿者工单系统（完善匹配、SLA、评价）
- 内容版本与更新提醒
- 更强的反垃圾与相似度去重
- 多语言协作翻译与复核

### 高风险功能（后置或严格受控）

- 房源交易/转租撮合（诈骗风险高，若做需强 KYC/押金托管/风控）
- 站内私信（若上线需强审计与反骚扰机制）
