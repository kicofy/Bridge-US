# 路线图与里程碑（建议 12 周 MVP）

## 阶段 0：准备（第 0-1 周）

- 明确 MVP 范围与“高风险不做清单”（如房源交易撮合）
- 信息架构定稿（城市/学校 + 主题 + 场景）
- 设计线框与基础视觉规范
- 内容模板（住房/诈骗/交通/超市）字段定稿

交付物：

- 原型与页面清单
- 数据模型草案
- 审核与治理规则草案

## 阶段 1：内容闭环（第 2-4 周）

- 登录与角色（user/contributor/moderator）
- 内容发布：指南/经验帖（模板化字段）
- 基础审核队列（待审/通过/驳回/需补充）
- 搜索与筛选（城市/学校/主题）
- 举报与基础反垃圾（频控、外链策略）

交付物：

- 可发布内容 + 可审核 + 可举报 的闭环
- 住房/诈骗专区 MVP

## 阶段 2：AI 问答 MVP（第 5-6 周）

- 构建“已审核内容”知识库（向量化 + 元数据）
- RAG 问答：答案 + 来源引用 + 可信状态 + 风险提示
- 问答入口与交互（澄清问题、保存/分享、纠错入口）

交付物：

- 可用的 AI 问答（只基于已审核内容）
- 回答日志与基本监控

## 阶段 3：AI 审查与风控增强（第 7-8 周）

- AI 内容初筛：违法/隐私/广告/诈骗标签
- 自动脱敏建议与作者修改引导
- 高风险内容强制人工复审（住房转账/联系方式/非官方外链）

交付物：

- 机器初筛 + 人工审核联动
- 风控命中率与误杀率看板（内部）

## 阶段 4：志愿者互助（第 9-10 周）

- 志愿者申请与审核
- 工单式求助（匿名、类别、时间窗、语言）
- 分配与 SLA、结束反馈

交付物：

- 可控的“真人支持”闭环
- 安全提示与边界声明

## 阶段 5：试点与运营（第 11-12 周）

- 校园/城市试点（种子内容 + 志愿者）
- 内容过期与更新机制上线
- 指标体系与增长实验（订阅标签、分享、推荐）

交付物：

- 试点版上线
- 一套可复用的运营 SOP（内容更新、审核 SLA、诈骗预警）
