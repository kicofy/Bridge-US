# 面向对象与用户场景

## 目标用户（Primary Users）

- **新到美国的国际学生（0–6 个月）**
  - 目标：快速建立“生存能力”（住房、交通、购物、医疗、安全）
  - 特征：信息不对称强、时间紧、容易焦虑、对诈骗不敏感

## 次级用户（Secondary Users）

- **在读国际学生/校友（贡献者）**
  - 目标：分享经验、避坑、帮助后来的同学
  - 特征：更了解本地语境、愿意提供经验但时间有限
- **志愿者（线上支持）**
  - 目标：提供答疑、陪伴式支持、协助核验与翻译
  - 特征：需要明确边界、流程与安全保障
- **版主/审核员（治理角色）**
  - 目标：保证内容质量、反诈骗、维护社区氛围
  - 特征：需要工具化的审核队列、证据/来源管理、申诉机制

## 典型用户旅程（User Journey）

### 旅程 A：落地第一周（高频）

- 需求：手机卡/银行卡/交通卡、怎么买菜、附近安全吗、如何通勤
- 行为：
  - 选择城市/学校 → 浏览“落地 7 天清单”
  - 询问 AI 快速答 → 看到来源与更新时间 → 保存/分享
  - 发现过期内容 → 一键提交“更新建议”

### 旅程 B：找房与入住（高风险）

- 需求：房源真假、区域治安、合同注意事项、押金/转租风险、骗局识别
- 行为：
  - 进入住房板块 → 筛选“预算/通勤/安全” → 阅读真实经验
  - 看到“已验证/部分验证”标记与证据
  - 需要真人帮助 → 发起“安全求助/志愿者咨询”工单

### 旅程 C：长期生活（持续价值）

- 需求：省钱攻略、医疗与保险、报税、驾照、校园资源、兼职规则
- 行为：
  - 订阅城市/学校标签 → 接收“重要更新提醒”
  - 贡献自己的经验 → 获得“有用度”与贡献积分

## 关键痛点与对应设计

- **信息真假难辨** → 来源引用 + 验证状态 + 更新日期
- **广告与营销干扰** → 禁止伪装建议的赞助内容 + 推广强制标注
- **不敢求助/怕泄露隐私** → 默认匿名展示 + 安全提示 + 受控沟通
- **内容过期** → 生命周期管理（过期提示、更新激励、版本记录）

## 板块分类（面向用户的导航语言）

平台建议以“板块（阶段/任务）”来组织入口，常用板块包括：

- 落地第一周
- 找房与入住
- 食物与超市
- 交通出行
- 办事与生活
- 安全与诈骗
