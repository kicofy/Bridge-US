# UI 设计规范（现代 / 简约 / 高级）

> 目标：做一个“看起来像产品而不是作业”的 UI：克制、留白、对比清晰、层级明确、动效轻量。  
> 默认英文开发（UI 文案可中英切换），视觉上避免“花/杂/重阴影”。

## 1) 视觉关键词

- **Modern**：干净网格、统一间距、轻阴影/玻璃质感点到为止
- **Minimal**：减少边框与线条数量，靠留白与层级建立结构
- **Premium**：高质量字体层级、柔和渐变背景、精致 hover 与 focus 状态

## 2) 色彩（深色为主，支持未来浅色扩展）

- Background：近黑偏蓝（减少刺眼）
- Surface：深色卡片 + 1px 低对比边框
- Accent：单一高饱和蓝作为主色（按钮/链接/强调）
- Text：主文字偏白，次文字偏灰蓝

建议变量（示例）：

- `--bg-0` / `--bg-1`
- `--surface-0` / `--surface-1`
- `--border`
- `--text-0` / `--text-1`
- `--accent` / `--accent-2`
- `--shadow`

## 3) 字体与排版

- 字体：系统 UI 字体优先（Windows/Apple/Android），未来可选接入 Inter
- 标题层级：
  - H1：32–40px（页面主标题）
  - H2：18–22px（卡片标题/模块标题）
  - Meta：12–13px（标签/时间/语言）
- 正文：
  - 行高 1.6
  - 最大宽度（阅读宽度）建议 72ch 左右

## 4) 布局与栅格

- 容器宽：`max-width: 1100px`
- 模块间距：24px
- 卡片圆角：14–18px
- 列表：统一用“卡片列表”而不是表格

## 5) 组件规范（MVP 必须一致）

- **Topbar**：透明/半透明 + blur，滚动不跳动
- **Buttons**：
  - Primary：accent 填充
  - Secondary：surface + border
  - Ghost：纯透明 + hover 背景
- **Cards**：
  - 默认 surface + border + 轻阴影
  - hover 提升边框对比/轻微上浮
- **Section cards**（分类块）：
  - 标题 + 简短说明（可选）
  - icon 可后置（先用简单字母/emoji 也行，但建议无 emoji）
- **Post cards**（帖子卡片）：
  - Title（可点击）
  - Meta（section · city · lang）

## 6) 页面结构（你现在的 4 个页面）

### Home（/）
- Hero：标题 + 一句话定位 + 主 CTA（Browse sections）
- Feed：信息流卡片（头像/作者/时间/标题预览/互动按钮）
- Recommended：第一版可用“最新 approved”当推荐（热度后置）
- Secondary：分类入口卡片（可只放按钮引导到 /sections）

### Sections（/sections）
- 网格：全部板块卡片
- 点击进入：`/sections/<code>`

### Section posts（/sections/<code>）
- 标题：板块名
- 列表：该板块 Feed（与首页一致）

### Post detail（/posts/<id>）
- 标题 + meta
- 内容排版（最大阅读宽度）
  - meta 建议包含作者信息（name/handle）、板块、城市、语言

## 7) 可用性（必须）

- Focus 可见（键盘可用）
- 可点击区域足够大（>= 40px）
- 对比度达标（至少接近 WCAG AA）
